# Purpose

This document defines how search should work across the knowledge base.

The search experience should help charity staff find trustworthy answers quickly, even if they do not know the correct AI, legal or technical terminology.

> Search should understand how charities ask questions, not how experts name concepts.

---

# Search Principles

Search should be:

- fast
- forgiving
- plain English
- task-based
- accessible
- privacy-conscious
- evidence-led

Navigation remains the primary way to browse the site. Search complements navigation rather than replacing it.

---

# Primary Search Goals

Users should be able to find answers to questions like:

- Can I use ChatGPT at work?
- Can I upload meeting notes?
- Do I need an AI policy?
- What is personal data?
- Can volunteers use AI?
- What should trustees know?
- Can AI help write funding bids?
- What does Green, Amber and Red mean?

---

# Search Sources

The search index should include:

- page titles
- summaries
- headings
- body text
- glossary terms
- FAQ entries
- page tags
- audience tags
- charity-size tags
- tool names
- workflow names

Do not index unpublished drafts or confidential material.

---

# Plain-English Synonyms

Search should understand common alternatives.

Examples:

- beneficiary = service user
- donor = supporter
- board = trustees
- HR = people team
- meeting notes = minutes
- AI = artificial intelligence
- LLM = large language model
- GDPR = data protection

Maintain the synonym list as terminology evolves.

---

# Ranking Priorities

Results should prioritise:

1. Before You Paste
2. Practical guidance
3. Charity workflows
4. Templates
5. Tool guides
6. Glossary
7. Governance
8. Research

High-risk guidance should rank above general explanations where appropriate.

---

# Zero-Result Searches

Track searches that return no useful result.

Review monthly to determine whether to:

- add synonyms
- improve titles
- create redirects
- write new content
- expand the FAQ

---

# Search Analytics

Monitor:

- most common searches
- failed searches
- search exits
- result clicks
- searches leading to feedback
- searches leading to downloads

Use trends to improve the knowledge base.

---

# Search Accessibility

Search must:

- support keyboard navigation
- announce results correctly to screen readers
- provide clear focus states
- avoid colour-only indicators
- remain usable on mobile devices

---

# Content Optimisation

Every page should include:

- a clear title
- a short summary
- descriptive headings
- meaningful keywords used naturally
- links to related guidance

Avoid keyword stuffing.

---

# Monthly Review

Each month review:

- failed searches
- confusing terms
- new AI products
- new regulatory language
- emerging charity questions

Update the search index and synonym list where required.

---

# Claude Prompt

```text
Review internal search behaviour.

Identify:

- failed searches
- confusing terminology
- missing synonyms
- pages that should rank higher
- opportunities for new FAQ entries

Recommend changes that improve findability without creating duplicate content.
```

---

# Success Criteria

Search succeeds when:

- users find answers in one or two searches
- non-technical language works well
- failed searches reduce over time
- search reinforces trust through accurate, relevant results
