# Purpose

This guide helps UK charities use Microsoft Copilot safely, confidently and proportionately.

Read it alongside the Before You Paste Framework, GDPR Decision Rules, Safeguarding Guidance and AI Tool Evaluation Framework.

# What Is Microsoft Copilot?

Microsoft Copilot is a family of AI tools available across Microsoft products and services.

Depending on the product, plan and account type, it may support:

- writing
- summarising
- meeting assistance
- email drafting
- spreadsheet analysis
- presentations
- search
- coding
- workflow support

Do not assume every Copilot product or plan has the same controls.

# Best Uses for Charities

Suitable uses may include:

- public website copy
- newsletters
- first-draft presentations
- meeting agendas
- generic policy drafts
- routine emails
- public report summaries
- non-sensitive spreadsheet support
- approved coding and documentation work

# Green Uses

Usually suitable with normal human review:

- public information
- fictional examples
- published reports
- brainstorming
- plain-English rewrites
- generic templates
- non-sensitive presentation drafts

# Amber Uses

Pause and check before using Copilot with:

- internal documents
- unpublished strategy
- internal meeting notes
- aggregated survey results
- draft board papers
- connected Microsoft 365 content
- confidential but non-personal information

Amber uses may require:

- approved organisational licensing
- tenant controls
- manager approval
- data-protection review
- appropriate permissions
- meaningful human review

# Red Uses

Do not use Copilot casually with:

- safeguarding records
- identifiable beneficiary information
- donor databases
- payroll
- grievance or disciplinary records
- medical information
- passwords
- API keys
- privileged legal advice
- sensitive incident reports
- confidential trustee disputes

# Microsoft 365 Context

Copilot may interact with:

- Outlook
- Word
- Excel
- PowerPoint
- Teams
- SharePoint
- OneDrive

Before enabling use, review:

- user permissions
- SharePoint and OneDrive access
- external sharing
- group membership
- transcript access
- retention
- admin controls
- audit capability

Poor permissions can become a wider AI risk.

# Permission Hygiene

Before rollout:

1. Review access to sensitive locations.
2. Remove unnecessary permissions.
3. Check external sharing.
4. Review dormant accounts.
5. Check Teams and group membership.
6. Identify sensitive folders.
7. Review transcript access.
8. Confirm who owns administration.

# Meetings and Teams

Copilot may help with:

- summaries
- action lists
- follow-up drafting

Pause before using it for meetings involving:

- safeguarding
- health information
- HR cases
- legal advice
- complaints
- confidential trustee matters

Check participant awareness, policy, retention and access.

# Word and Outlook

Suitable uses:

- rewriting public text
- improving clarity
- drafting routine emails
- summarising non-sensitive threads

Use caution with:

- complaints
- legal matters
- employment
- donor disputes
- vulnerable service users
- confidential contracts

# Excel and Data Analysis

Suitable uses:

- formula help
- chart suggestions
- aggregated analysis
- formatting support

Do not use sensitive spreadsheets casually.

Examples:

- payroll
- beneficiary records
- donor histories
- employee information
- identifiable survey exports

# PowerPoint

Suitable uses:

- first-draft presentations
- public report summaries
- trustee briefing structures
- adapting content for audiences

Always review facts, images, citations, accessibility and confidential content.

# Human Review

Always check:

- accuracy
- bias
- tone
- accessibility
- source support
- confidentiality
- permissions
- whether the output affects a person

# Common Mistakes

Avoid:

- assuming Microsoft branding makes every use safe
- confusing personal and organisational accounts
- assuming every plan has the same safeguards
- enabling Copilot before fixing permissions
- using transcripts without approval
- sending AI-written emails without review
- uploading sensitive spreadsheets
- assuming tenant controls replace governance

# Governance Requirements

Define:

- approved Copilot products
- approved licences
- approved users
- permitted use cases
- prohibited information
- admin responsibility
- meeting rules
- human-review requirements
- incident reporting
- training
- review frequency

# Charity Size Guidance

## Small Charities

Start with:

- one approved organisational product
- limited users
- low-risk drafting only
- do-not-paste rules
- one named owner
- simple staff training

## Medium-Sized Charities

Add:

- departmental use cases
- role-based permissions
- AI-use register
- procurement and data-protection review
- trustee oversight

## Large or Complex Charities

Add:

- formal tenant governance
- sensitivity labels
- audit and monitoring
- legal and security review
- DPIAs where needed
- integration governance

# Review Questions

Ask:

- Which Copilot product are we using?
- Which licence applies?
- Is the account organisational?
- What can the user access?
- Are permissions appropriate?
- Is personal data involved?
- Is safeguarding information involved?
- Could the output affect someone?
- Is human review meaningful?
- Are staff trained?

# Review Triggers

Review when:

- products change
- licensing changes
- privacy documentation changes
- Microsoft 365 integration changes
- Teams features change
- regulator guidance changes
- a material security incident occurs

# Success

This guide succeeds when charities:

- understand that Copilot products differ
- recognise that Microsoft 365 permissions matter
- know which tasks are low risk
- know what information requires caution
- keep meaningful human review
- avoid treating workplace integration as automatic safety
