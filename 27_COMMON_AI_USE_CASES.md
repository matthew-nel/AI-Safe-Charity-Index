# Purpose

This document defines the standard AI use cases that appear throughout the knowledge base.

It gives charities practical, task-based guidance rather than abstract AI advice.

Every use case should answer:

- Can AI help?
- Is it safe?
- What are the risks?
- What data should never be used?
- What checks should happen before publishing or acting?

# Standard Use Case Structure

Each page should include:

1. Goal
2. Suitable AI tools
3. Green examples
4. Amber examples
5. Red examples
6. GDPR considerations
7. Human review requirements
8. Safer alternatives
9. Related workflows
10. Source IDs and Claim IDs

# Core Use Cases

## Writing and Editing
- Draft articles
- Rewrite plain English
- Summarise reports
- Improve accessibility
- Generate headlines

Green:
- Public information
- Fictional examples

Red:
- Identifiable beneficiary stories
- Confidential reports

## Meeting Notes
- Summarise meetings
- Produce actions
- Draft minutes

Never upload:
- Safeguarding discussions
- HR investigations
- Legal advice
- Sensitive board discussions
without approved tools and governance.

## Fundraising

Use AI to:

- Brainstorm campaigns
- Draft grant applications
- Improve donor communications

Never upload:
- Donor databases
- Payment information
- Giving histories

## Research

Suitable for:

- Literature summaries
- Theme identification
- Survey design

Not suitable for:

- Publishing AI-generated findings without verification
- Uploading identifiable research participants

## Communications

Suitable for:

- Social media
- Newsletter drafts
- Website copy
- Press release structure

Always:

- Fact check
- Check tone
- Check accessibility
- Review by a human

## HR

Suitable for:

- Job descriptions
- Interview questions
- Training material

Never:

- Rank candidates automatically
- Upload employee records
- Make disciplinary decisions

## Governance

Suitable for:

- Agenda drafting
- Policy summaries
- Board paper structure

Never:

- Delegate trustee decisions
- Upload legally privileged advice

## Coding and Digital

Suitable for:

- HTML
- CSS
- Documentation
- Accessibility suggestions
- SQL explanations

Always:

- Test code
- Review security
- Remove secrets
- Protect production systems

# Traffic Light Summary

## Green

- Public information
- Fictional content
- Generic drafting
- Brainstorming
- Translation
- Plain-English editing

## Amber

- Internal information
- Anonymised material
- Draft strategies
- Internal reports
- Meeting summaries

Requires judgement and approved tools.

## Red

- Personal data
- Special category data
- Safeguarding information
- Payroll
- HR records
- Donor databases
- Passwords
- API keys
- Legal advice
- Confidential contracts

Do not upload to public AI tools.

# Cross-Cutting Rules

Always ask:

- Is this personal data?
- Is this confidential?
- Can it be anonymised?
- Is AI actually needed?
- Does a human review the output?
- Could an error cause harm?
