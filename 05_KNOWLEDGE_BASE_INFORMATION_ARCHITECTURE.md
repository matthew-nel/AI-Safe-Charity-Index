# AI Safe Use Knowledge Base for UK Charities

## Knowledge Base Information Architecture

**Version:** 1.0.0 **Status:** Foundation Document **Owner:** Matthew
Nel **Last reviewed:** July 2026 **Next review:** October 2026

# Purpose

This document defines the information architecture for the public GitHub
knowledge base.

The site should help people find practical answers quickly, regardless
of their technical confidence.

# Design Principles

-   Navigation should be task-based rather than policy-based.
-   Prioritise small and medium-sized charities.
-   Keep journeys short.
-   One question per page where possible.
-   Allow deeper exploration for larger organisations without
    complicating the default experience.

# Homepage

The homepage should answer:

-   What is this?
-   Who is it for?
-   Where should I start?
-   What has changed recently?

It should prominently feature the "Before You Paste" framework.

# User Journeys

Every page should end with:

-   Related guidance
-   Relevant templates
-   Next recommended page
-   Feedback link

# Scalability

The architecture should support hundreds of pages without restructuring.

New content should fit naturally within the existing navigation.

# Definition of Success

Users should be able to find the answer to their question within three
clicks and understand the next action within five minutes.
