# Contributing

Thank you for your interest in contributing to AI Safe Charity Index.

This project improves through charity experience, evidence, research, specialist review, public feedback, practical examples, corrections and open-source collaboration.

## Ways to Contribute

You can help by:

- reporting an error
- flagging outdated guidance
- suggesting a source
- asking a question
- suggesting a new topic
- sharing an anonymised charity workflow
- reviewing a page for accuracy, accessibility or plain English
- proposing a pull request

## Contribution Principles

Contributions should support:

- evidence before opinion
- small charities first
- plain English
- practical guidance
- human judgement
- safeguarding
- data protection
- transparency
- accessibility
- vendor neutrality
- proportionality

Do not include personal data, safeguarding case details, confidential organisational information, API keys, passwords or privileged legal advice.

## Evidence Standards

Material factual claims should use reliable sources. Preferred sources include legislation, regulators, government, official provider documentation, recognised charity-sector bodies, and academic or professional research.

Where evidence is uncertain, say so.

## Pull Requests

For larger changes, open an issue before preparing a pull request.

All pull requests are reviewed before merge. Higher-risk topics such as safeguarding, GDPR, employment, legal interpretation, automated decision-making and vulnerable people may require additional review.
