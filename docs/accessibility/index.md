# Accessibility

The project aims to meet WCAG 2.2 AA wherever practical.

Accessibility is part of editorial quality, technical quality and public trust. Guidance should use clear headings, descriptive links, plain English, accessible tables, meaningful alt text and text labels that do not rely on colour alone.

Green, Amber and Red guidance must always include text labels and explanations, not colour alone.
