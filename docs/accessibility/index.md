# Accessibility

The project aims to meet WCAG 2.2 AA wherever practical.

Accessibility is part of editorial quality, technical quality and public trust. Guidance should use clear headings, descriptive links, plain English, accessible tables, meaningful alt text and text labels that do not rely on colour alone.

Green, Amber and Red guidance must always include text labels and explanations, not colour alone.

## What We Aim To Do

- use clear headings and landmarks
- keep link text descriptive
- support keyboard navigation
- preserve visible focus states
- avoid colour-only meaning
- write in plain English
- keep tables usable on small screens
- provide text alternatives for meaningful images
- use captions or transcripts where media is added
- test important pages manually as well as with automated checks

## AI-Generated Content

AI-generated or AI-assisted content must be checked by a person before publication for:

- headings and structure
- factual accuracy
- plain-English clarity
- alt text
- captions and transcripts
- colour contrast
- bias, stereotypes and exclusion
- whether synthetic content could mislead people

Automated captions, translations, summaries and alt text should be treated as drafts.

## Known Limitations

The site is still in early development. Accessibility review should be repeated after major navigation, template, form or media changes.
