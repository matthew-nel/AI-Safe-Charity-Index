# Monitoring, Audit and Change Control

AI approvals can go stale quickly. Models, features, connectors, retention settings and provider terms change.

## What to Monitor

- new or changed use cases
- user access and offboarding
- prompts, files and outputs where logs are available
- connector permissions
- incidents, near misses and complaints
- human-review overrides
- accuracy, bias or safety issues
- provider terms, subprocessors and model changes
- DPIA and supplier-review dates

## Change Triggers

Reapprove or re-screen when there is:

- a new model
- a new account plan
- a new connector or agent capability
- a new data source
- a new purpose or affected group
- a new external action, such as sending or publishing
- a provider term, retention or training change
- a material incident or near miss

## Simple Monthly Check

For small charities, a light monthly check can ask:

1. Did anyone start using a new AI tool or feature?
2. Did any approved tool change settings, terms or models?
3. Were there any incidents, near misses or complaints?
4. Did any user leave, change role or no longer need access?
5. Are any Amber or Red uses missing review?
