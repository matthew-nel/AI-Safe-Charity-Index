# Purpose

This document defines every content type, template and reusable resource within the knowledge base.

The goal is consistency. Every page, template and downloadable resource should follow a predictable structure so charities know what to expect.

# Core Content Types

## 1. Practical Guide

Purpose:
Help someone complete a task safely.

Required sections:

- Quick answer
- Traffic-light result
- What to do
- Example
- Risks
- Related guidance
- Sources
- Version

## 2. Playbook

Purpose:
Provide an end-to-end workflow.

Examples:

- Using AI for fundraising
- Using AI in communications
- Using AI for trustees

Required sections:

- Goal
- Before You Paste
- Recommended workflow
- Common mistakes
- Templates
- Further reading

## 3. Explainer

Purpose:
Explain a concept in plain English.

Examples:

- Personal data
- Hallucinations
- Special category data
- DPIAs

Required sections:

- Plain English definition
- Charity example
- Why it matters
- Related guidance

## 4. Tool Review

Each AI tool page should include:

- Overview
- Best uses
- Risks
- Data considerations
- Questions to ask
- Official documentation
- Last reviewed

## 5. Template

Every downloadable template should include:

- Purpose
- When to use it
- Instructions
- Editable template
- Example
- Review date

# Standard Templates

The knowledge base should include:

- AI Policy
- Staff Guidance
- Trustee Briefing
- AI Tool Assessment
- AI Experiment Log
- AI Use Register
- Output Review Checklist
- Incident Report
- Procurement Checklist
- Risk Register
- DPIA Triage Checklist
- Meeting AI Checklist
- Before You Paste Poster

# Page Metadata

Every page should display:

- Version
- Last reviewed
- Next review
- Evidence level
- Primary sources
- Author
- Related pages

# Naming Standards

Use clear, task-based titles.

Prefer:

- Can I use AI for...
- How to...
- Before You Paste...
- AI for...

Avoid vague titles.

# Quality Checklist

Before publishing:

- Audience identified
- Plain English
- One problem solved
- Practical example included
- Sources checked
- Metadata complete
- Accessibility reviewed

# Definition of Success

Every resource should be immediately usable by a charity without requiring specialist knowledge or additional interpretation.
