# Purpose

The Claim Register tracks every important factual claim published in the knowledge base.

Each claim must be traceable to one or more sources.

Claims should be reviewed whenever supporting evidence changes.

# Core Principle

> Every important public statement should be supported by evidence.

# Claim ID

Recommended format:

CLAIM-0001

Never reuse or renumber claim IDs.

# Review Process

When a source changes:

1. Find linked claims.
2. Check whether the wording remains accurate.
3. Update affected pages.
4. Record the review.
5. Publish changes in the next release.

# New Claim Workflow

1. Identify the factual statement.
2. Link supporting source IDs.
3. Record confidence.
4. Add page references.
5. Schedule review.

# Unsupported Claims

If a claim cannot be supported:

- remove it
- rewrite it
- mark it as opinion
- identify research needed

Do not publish unsupported statements as fact.

# Monthly Prompt

Review every claim against the current Source Register.

Flag:

- outdated evidence
- conflicting evidence
- unsupported claims
- claims needing stronger sources
- pages affected

Produce a list of recommended updates.

# Success

The Claim Register succeeds when every important statement on the website can be traced back to reliable evidence.
