# Purpose

This document maps the knowledge base to the real work charities do.

The public resource should be organised around practical tasks and decisions, not only around AI, GDPR or governance concepts.

People usually arrive with a question such as:

- Can I use AI to write this funding application?
- Can I upload meeting notes?
- Can volunteers use ChatGPT?
- Can AI help analyse survey responses?
- Can AI summarise a beneficiary story?
- Can AI help trustees review a board paper?

The workflow model ensures Claude creates guidance that is useful in everyday charity work.

# Core Principle

> Start with the task the charity is trying to complete, then explain the risks, checks and safer workflow.

# Cross-Workflow Risk Questions

Every workflow should ask:

- Is personal data involved?
- Is special category data involved?
- Is safeguarding information involved?
- Is the information confidential?
- Is the tool approved?
- Is the account appropriate?
- Could the output affect a person?
- Could an error cause harm?
- Is human review meaningful?
- Is specialist approval needed?
- Can the task be completed with less data?
- Can fictional or public information be used instead?

# Workflow Scaling

## Small Charity

Use:

- clear rules
- one named owner
- a short approved-tools list
- simple templates
- low-risk use cases
- manager escalation

## Medium-Sized Charity

Use:

- role-based guidance
- formal tool approval
- AI-use register
- departmental ownership
- training
- quarterly review

## Large or Complex Charity

Use:

- system inventory
- procurement assurance
- DPIAs
- technical controls
- audit
- specialist legal and security review
- formal governance

Prefer task-based titles:

- Can I use AI to write a funding bid?
- Can AI summarise meeting notes?

- Can AI analyse survey responses?

# Workflow Development Process

When adding a workflow:

1. define the real task
2. identify the audience
3. identify the data involved
4. classify Green, Amber and Red uses
5. identify the tool requirements
6. identify human review
7. identify approval needs
8. provide safer alternatives
9. scale by charity size
10. link templates and evidence
11. test with charity users
12. review after feedback

# Workflow Research Questions

Use charity research to understand:

- which tasks are most common
- where staff are already using AI
- what feels unclear
- which teams need guidance
- where unsafe workarounds appear
- which templates reduce friction
- what differs by charity size
- what differs by subsector

# Success Criteria

The workflow system succeeds when charity staff can:

- find guidance based on the task they are doing
- understand the risk quickly
- see a realistic example
- follow a safer process
- know when to stop
- know who to ask
- use a relevant template
- find deeper guidance when needed

The knowledge base should feel organised around charity work, not around AI jargon.
