# Purpose

This document maps the entire AI Safe Use Knowledge Base for UK Charities.

It shows how:

- audiences
- charity sizes
- workflows
- guidance
- templates
- evidence
- research
- governance
- releases
- technical systems

connect to one another.

The Knowledge Map helps prevent:

- duplicated pages
- missing guidance
- disconnected templates
- orphaned evidence
- unclear learning paths
- inconsistent risk decisions
- navigation that reflects internal structure rather than user needs

The core principle is:

> **Every page should have a clear purpose, audience, evidence base and place in the wider knowledge system.**

# Project Mission

The central mission is:

> **Help charities use AI safely, confidently and responsibly through practical, evidence-led guidance they can trust.**

The product promise is:

> **Help charities save time with AI without risking people's data, trust or safety.**

Every content area should support at least one part of this mission.

# Duplicate Content Rules

Do not create a new page when:

- an existing page can be improved
- a learning path can link to existing guidance
- a role page can curate existing content
- a template can be reused
- an FAQ can answer the question briefly

Create a new page when:

- the user task is distinct
- the risk is materially different
- the guidance would otherwise become too long
- search data shows a separate need
- the audience requires a different action

---

# Orphaned Knowledge

A knowledge item is orphaned when it is not connected to the wider system.

Examples:

- a source with no claims
- a claim with no page
- a page with no navigation or related links
- a template with no guidance
- a research finding with no impact
- a roadmap item with no owner
- a tool review with no next-review date

Review orphaned knowledge monthly.
